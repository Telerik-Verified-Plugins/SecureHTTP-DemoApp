Secure HTTP Plugin Demo

The Secure HTTP plugin allows you to securely communicate with your endpoinst. In addition, it allows you to offload HTTP(S) requests to background threads through native code.

This plugin helps you connecting to Basic Authentication protected resources, as well as Pinning to SSL Certificates.

Custom plugins don't work in the simulator and the AppBuilder companion app, so inside Telerik AppBuilder choose Run - Build - iOS/Android - App package.

SSL Pinning

When enabled, the endpoint of our HTTPS requests is checked for a valid certificate according to the .cer files stored in the www/certificates folder.

HTTPS GET requests

Normally requests are executed by the webview. This plugin offers native connections which run in background threads. If SSL Pinning is enabled, only connections to trusted endpoints are allowed.

Basic Authentication

If your endpoint is protected by Basic Authentication you can globally set the correct header.
